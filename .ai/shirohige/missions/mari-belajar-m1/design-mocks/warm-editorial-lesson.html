Cara Kerja Komputer — Dari Bit sampai Program Berjalan

Basic Intermediate Advanced • 14 menit baca

Bagian 1: Dari Listrik ke Bit

Komputer pada dasarnya adalah mesin yang memproses informasi. Semua informasi pada akhirnya harus diubah ke bentuk yang bisa dipahami oleh mesin: bit.

Bit adalah unit terkecil informasi di komputer. Bit hanya memiliki dua kemungkinan nilai: 0 atau 1. Anggap saja bit seperti saklar lampu — lampu bisa mati (0) atau menyala (1).

Karena satu bit terlalu kecil, komputer mengelompokkan bit menjadi byte. Satu byte terdiri dari 8 bit. Dengan 8 bit, kita bisa merepresentasikan 28 = 256 kemungkinan nilai berbeda.

Nilai 65 dalam ASCII adalah huruf A. Jadi, byte 01000001 bisa berarti angka 65, atau huruf A, tergantung bagaimana program membacanya.

Contoh: Mengonversi Desimal ke Biner dengan JavaScript

Di JavaScript, kita bisa memanfaatkan method .toString(2) untuk mengubah angka desimal menjadi string biner.

JavaScript 
const decimal = 65;
const binary = decimal.toString(2).padStart(8, "0");

console.log(binary); // "01000001"

// Operasi bitwise AND untuk memeriksa bit tertentu
const mask = 0b00001000; // bit ke-3 (nilai 8)
const hasBit = (decimal & mask) !== 0;

console.log(hasBit); // false

Bagian 2: Bagaimana Data Direpresentasikan

Komputer menyimpan angka sebagai byte-byte di memori. Selain biner, programmer sering menggunakan heksadesimal (basis 16) karena lebih ringkas.

  • Desimal 255 = Biner 11111111 = Heksadesimal 0xFF
  • ASCII menggunakan 1 byte per karakter
  • Unicode bisa membutuhkan beberapa byte per karakter

Compiler, Interpreter, dan Transpiler

Sebelum program bisa berjalan, kode yang ditulis programmer harus diubah menjadi instruksi mesin. Ada beberapa cara: compiler menerjemahkan seluruh kode sebelum dijalankan, interpreter menjalankan baris per baris, dan transpiler menerjemahkan antar bahasa tingkat tinggi.

Bagian 3: Dari Instruksi sampai Program Berjalan

CPU mengeksekusi instruksi mengikuti siklus fetch-decode-execute. Siklus ini berlangsung sangat cepat — miliaran kali per detik pada prosesor modern.

Ketika program berjalan, sistem operasi memberinya area memori yang dibagi menjadi code segment, data segment, stack, dan heap. Memahami layout memori membantu programmer memahami mengapa variabel lokal hilang setelah fungsi selesai.
